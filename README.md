# Пояснительная записка к тестовому заданию
**Кандидат:** Казанцев Кирилл 
**Вакансия:** C# (ASP.NET) Developer в компанию Сайберс  
**Продукт:** Система управления проектами (Project Management System)  

Данный проект представляет собой законченное клиент-серверное веб-приложение для ведения базы данных проектов, сотрудников и связанных задач, архитектурно разделенное на слои и готовое к работе.

---

## 1. Технологический стек и версии пакетов

### Backend
- **Платформа:** .NET 10.0
- **База данных:** SQLite (локальный файл, разворачивается автоматически)
- **Архитектура:** Трёхкомпонентная архитектура (Data Access, Business Logic, Presentation) с паттерном DTO и валидацией на уровне приложения.

**Использованные NuGet-пакеты:**
* `Microsoft.EntityFrameworkCore.Sqlite (10.0.9)` — легковесная СУБД для простоты развертывания проекта без настройки внешних серверов баз данных.
* `Microsoft.AspNetCore.OpenApi (10.0.9)` + `Scalar.AspNetCore (2.16.7)` — генерация спецификации OpenAPI и современный интерактивный интерфейс документации API.
* `AutoMapper.Extensions.Microsoft.DependencyInjection (12.0.1)` — автоматическое маппинг-преобразование сущностей БД в DTO.
* `FluentValidation.AspNetCore (11.3.1)` — строго типизированная валидация входящих запросов изолированно от моделей.
* `Microsoft.EntityFrameworkCore.Design (10.0.9)` — инструменты поддержки миграций базы данных.

### Тестирование
Бизнес-логика бэкенда покрыта юнит-тестами с использованием следующих инструментов:
* `xunit (2.9.3)` + `xunit.runner.visualstudio (3.1.4)` — среда для выполнения тестов с изоляцией состояния экземпляров.
* `FluentValidation.Validators.UnitTestExtension (1.12.0.5)` — расширение для тестирования правил валидации.
* `Microsoft.NET.Test.Sdk (18.7.0)` + `coverlet.collector (6.0.4)` — тестовое окружение и сбор метрик покрытия кода.

### Frontend
Интерфейс представляет собой Single Page Application (SPA), разработанное на современном стеке:
- **Базовые технологии:** React 19.2, TypeScript 6.0, сборщик Vite 8.1.
- **Интерфейс и UI-компоненты:** Material UI (MUI v9) для обеспечения адаптивности, включая таблицы `@mui/x-data-grid` и календарные компоненты `@mui/x-date-pickers`.
- **Работа с формами и валидация:** `react-hook-form` совместно с валидатором `zod` и интеграционным адаптером `@hookform/resolvers`.
- **Загрузка файлов:** `react-dropzone` для реализации HTML5 File Uploader с поддержкой Drag & Drop документов.
- **Взаимодействие с API:** `axios` для асинхронных AJAX-запросов и маршрутизатор `react-router-dom` (v7).
- **Линтинг и качество кода:** `oxlint` для высокоскоростного статического анализа кода фронтенда.

## 2. Инструкция по конфигурации и запуску

### Требования к окружению
- Установленный **.NET 10 SDK**
- Установленная среда выполнения **Node.js** (v18 или выше)

### Порядок запуска

1. Откройте терминал в корневой папке решения и запустите backend-приложение:
   ```bash
   dotnet run --project ProjectManagement.API
   ```
   *Проект скомпилируется, автоматически создаст локальный файл базы данных SQLite, применит миграции и поднимет Web API на портах по умолчанию.*

2. Откройте второй терминал, перейдите в папку фронтенда, установите зависимости и запустите сервер разработки Vite:
   ```bash
   cd Frontend
   npm install
   npm run dev
   ```

3. Если вам необходимо быстро наполнить пустую базу данных демонстрационными сотрудниками и проектами, запустите консольный скрипт сидинга в папке scripts проекта:
   ```bash
   node seed.js
   ```
   *Скрипт автоматически создаст тестовых сотрудников, распределит их по ролям (руководители/исполнители) и добавит связанные проекты.*


4. Перейдите по адресу `http://localhost:3000`, чтобы открыть интерфейс программы.

5. Чтобы изучить и протестировать эндпоинты API, откройте в браузере документацию Scalar по адресу:  
   `https://localhost:5119/scalar/v1`

### Запуск Unit-тестов
Для проверки работоспособности бизнес-логики и прохождения тестов выполните в корне решения команду:
```bash
dotnet test
```
